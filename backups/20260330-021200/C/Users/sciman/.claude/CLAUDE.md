## 1. 阅读指引（必读）
- 本文件定义跨仓通用规则语义（WHAT）；仓库根目录同名文件定义本仓落地动作（WHERE/HOW）。
- 固定结构：`1 / A / B / C / D`。
- 裁决链：`运行事实/代码 > 项目级文件 > 全局文件 > 临时上下文`。

## A. 共性基线（全局）
### A.1 三层职责（强制）
- 共性基线：统一执行与治理标准（WHAT）。
- 平台差异：仅写 Claude 特有加载/诊断/回退（PLATFORM）。
- 项目差异：仅在项目级文件落地仓库事实（WHERE/HOW）。

### A.2 执行与输出
- 默认中文沟通；代码/命令/日志/报错保持英文原文。
- 简单任务：`Result + Evidence`；复杂任务：`Goal / Plan / Changes / Verification / Risks`。
- 默认持续执行到完成；仅在真实阻塞、不可逆风险、连续自修复失败时请求人工确认。

### A.3 强制规则（R1-R8）
1. `R1 先定归宿再改动`：先声明当前落点与目标归宿。
2. `R2 小步闭环`：每步可执行、可验证、可对比。
3. `R3 根因优先`：止血补丁必须标注回收时点与最终归宿。
4. `R4 风险分级`：低风险自动执行；中风险发布前确认；高风险先预演回滚。
5. `R5 反过度设计`：无证据不做预抽象和猜测式优化。
6. `R6 硬门禁`：`build + test + contract/invariant + hotspot` 不可绕过。
7. `R7 一致性与兼容`：未授权不得破坏契约、数据格式、外部行为与向后兼容。
8. `R8 可追溯`：每次变更必须留存 `依据 -> 命令 -> 证据 -> 回滚`。

### A.4 N/A 策略
- 仅纯文档/纯注释/纯排版可将 `R6` 标记 `N/A`。
- 必须写明理由与替代验证，不得整项跳过。

### A.5 治理演进（E1-E6）
- `E1` 版本化；`E2` 兼容窗口；`E3` Waiver（`owner/expires_at/status/recovery_plan/evidence_link`）；`E4` 健康指标；`E5` 供应链门禁；`E6` 数据结构变更治理。

## B. Claude 平台差异（全局）
### B.1 加载链与覆盖
- 文件名优先级：`CLAUDE.override.md > CLAUDE.md > fallback`。
- 推荐用户目录：`~/.claude`；以本机 CLI 实际加载结果为准。
- `CLAUDE.override.md` 仅用于短期排障；任务结束后删除并复测。

### B.2 最小诊断动作
1. `claude --version`（记录运行时版本）。
2. `claude --help`（核对子命令可用性）。
3. 若存在状态子命令，再执行并留痕；若不存在，以 `--help` 输出作为状态证据。

### B.3 不支持项回退
- 命令缺失或行为不一致时，必须记录：`N/A`、原因、替代命令、证据位置。

## C. 项目级承接契约（全局模板）
### C.1 自包含与边界
- 项目级同名文件必须完整自包含，并显式承接 `GlobalUser/CLAUDE.md`。
- 项目级仅写本仓事实，不复述全局 R/E 正文。

### C.2 项目级必填项
- 模块边界与归宿。
- 门禁命令与顺序：`build -> test -> contract/invariant -> hotspot`。
- 失败分流与阻断条件。
- 证据与回滚位置。
- `Global Rule -> Repo Action` 承接映射。

### C.3 协同目标
- 达成“全局定标准、项目定动作”，避免重叠与缺失，实现 1+1>2 协作效能。

## D. 维护校验清单（全局）
- 结构必须保持 `1 / A / B / C / D`。
- 全局文件不得写仓库特有路径与命令。
- 三层协同完整：`共性基线 + 平台差异 + 项目差异`。
- 协同链完整：`规则 -> 落点 -> 命令 -> 证据 -> 回滚`。
- 升级后需校验与项目级文件版本联动与承接映射一致性。
